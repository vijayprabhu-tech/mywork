Project : BnkLedgerSpringBootH2
Desc: Idempotent Background Transaction Processing Service.

//////////////////// Pre-requisite /////////////////
Tech Stack: Java 17, Springboot 3x, Webservice, Tomcat, H2, JPA 3x, Redis 5x, Git, IntelliJ, postman

Requirements covered:
1) 2 Api call, one for session id creation, other for submitting the payment.
2) Achieves idempotent request by using Idempotency-Key in the header and Session id.
3) if validation on both keys got success, then system will insert the record of
  	Receiver, Amount, PaymentType, TransactionId along with SessionId in projects.payments table.
4) if validation got failed on both invalid (Session id / Idempotency-Key): system will proceed with the appropriate error messages on the payment response and not getting any effect on the projects.payments table.


//////////////////// Local Run Instruction ///////////////
1. configure java17 in project setting
2. check out the project (BnkLedgerSpringBootH2) code from the below given git repo, make the maven compile with error free.
3. Start the redis server.
4. start the tomcat app server on "com.bnk.ecomproj.ProjApplication" thru edit config.
5. thru the postman, trigger the first Api, it will generate the paymentSessionId. Check the H2 db of projects.payment_sessions table. It should have the valid record of paymentSessionId data.
6. copy the session id and paste it in the second Api payload(paymentSessionId) and Add header key value as "Idempotency-Key  : ik_1". After trigger the 2 api call, Check the H2 db of projects.payments table. It should have the valid payment record with transactionId.
